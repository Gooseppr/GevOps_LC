# Concepts Cloud & Modèles AWS

## Objectifs pédagogiques

- Comprendre les modèles de service cloud (IaaS, PaaS, SaaS)
- Identifier les composants fondamentaux d’une infrastructure AWS
- Distinguer région, AZ et edge locations
- Expliquer le modèle de responsabilité partagée AWS
- Comprendre les notions de scalabilité et haute disponibilité

## Définition

Le cloud computing est un modèle de fourniture de ressources informatiques à la demande via Internet, permettant d’éviter la gestion physique de serveurs.

## Pourquoi ce concept existe

Avant le cloud :
- Achat de serveurs coûteux
- Provisionnement lent
- Maintenance complexe

Le cloud apporte :
- Rapidité
- Scalabilité
- Paiement à l’usage

## Fonctionnement

IaaS / PaaS / SaaS définissent le niveau de responsabilité.

## Résumé

Le cloud simplifie l’infrastructure et permet de scaler rapidement avec un modèle économique flexible.
